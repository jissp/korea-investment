# 작업 지침

## 1. 모듈 분석 대상을 확인합니다.

1. 사용자가 명시적으로 특정 모듈의 분석을 요청한 경우 대상을 해당 모듈로 한정하세요.
2. 만약 사용자가 명시적으로 대상을 전달하지 않았다면 더이상 분석하지 말고 종료하세요.

## 2. 모듈 분석

1. 해당 모듈에 대해서 파일 구조, 비지니스 로직, 흐름을 확인하고 이해하세요.
2. 해당 모듈이 어떤 기능을 담당하고 있고, 사용 예제 등을 숙지하세요.

## 3. 모듈의 README.md 파일 작성

1. 모듈의 README.md 파일은 architecture-template.md 파일을 참고하여 작성하세요.
2. README.md 파일은 해당 모듈 디렉토리 내부에 위치해야 합니다.
